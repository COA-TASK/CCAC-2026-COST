# CCAC-2026-COST
# 复杂观点分析任务

## 任务一：跨文档事件-观点联合抽取

### 任务背景

跨文档事件 - 观点联合抽取（Cross-Document Event-Opinion Extraction, CodEOE）旨在理解社交媒体场景下跨文档长文本中事件与观点的内在关联，联合抽取事件结构化信息与对应的观点情感内容，在舆情监测、公共事件舆论研判、社交平台内容治理、热点事件公众态度分析等场景具有广泛的应用。与传统的单文档事件抽取、属性级细粒度情感分析任务不同，跨文档事件 - 观点联合抽取关注于对事件本身与公众观点情感间因果关联的深度理解，突破了现有研究将事件信息抽取与情感观点分析割裂开展、仅面向单文档短文本、局限于固定领域属性级分析的诸多局限。该任务需要充分挖掘跨文档语境中事件与观点的语义关联，建模分散在多篇文档中的事件触发词、论元要素与对应观点表述、情感极性之间的映射关系，进而联合抽取完整的事件触发词 - 论元对与事件 - 观点 - 情感三元组。本基准任务鼓励研究者从跨文档长上下文语义建模、事件与观点的联合交互学习、结构化信息与情感倾向的联合解码等不同角度，对社交媒体跨文档场景下的事件级观点情感进行建模与抽取，旨在推动自然语言处理领域中信息抽取与情感分析两大核心方向的交叉融合与技术发展。

### 任务介绍

CodEOE 全称为 Cross-Document Event-Opinion Extraction（跨文档事件-观点联合抽取），实现了跨文档场景下事件抽取与事件级情感分析的端到端联合建模，填补了两大核心任务交叉研究的空白，专为社交媒体热点事件舆情分析、公众观点挖掘的真实场景设计。

### 任务定义

给定一个由1篇热点新闻主文档 + N条与该新闻强相关的社交媒体用户评论文档组成的跨文档文本集合，要求模型通过理解跨文档全局长上下文语义，端到端联合抽取两类结构化信息，同时完成事件结构解析与事件级观点情感挖掘两大核心目标：

1. **事件结构抽取**：识别文本中所有的事件触发词，匹配与每个触发词有直接语义绑定的事件论元，生成所有合法的事件触发词-论元对（Trigger-Argument Pairs, Tri-Arg）；
2. **事件级观点情感抽取**：识别文本中所有针对特定事件的主观观点内容，匹配观点对应的事件触发词，标注观点针对该事件的情感极性，生成所有合法的事件触发词-观点-情感三元组（Trigger-Opinion-Sentiment Triplets, Tri-Opi-Senti）。

任务二：嵌套观点六元组抽取
任务背景
嵌套观点六元组抽取（Nested Opinion Sextuple Extraction, NOSE）旨在深度解析自然文本中层级化的观点表达与嵌套式的态度立场，在社交媒体舆情分析、电商评论深度挖掘、政策民意研判、智能人机交互等自然语言处理核心场景具有广泛的应用价值。与传统的观点角色标注、结构化情感分析任务不同，嵌套观点六元组抽取突破了句子级扁平化观点抽取的局限，聚焦于对文本中“观点的观点”这一嵌套语义现象、二阶态度立场以及文档级跨句共指关联的深度理解与完整建模。该任务需要精准识别文本中的一阶基础观点四元组，同时挖掘针对该基础观点的二阶态度要素，建模两层观点之间的嵌套语义依赖关系，最终实现完整嵌套观点六元组的精准抽取。本任务提出了全新的文档级嵌套观点分析研究范式，构建了配套的中英双语基准数据集与端到端基线模型，旨在填补现有嵌套观点分析领域的研究空白，推动观点分析任务从句子级单一层级的要素抽取，向文档级复杂层级的语义深度理解方向发展，为细粒度情感语义理解、复杂舆情研判等相关研究提供新的基准支撑与技术思路。

任务介绍
NOSE 全称为 Nested Opinion Sextuple Extraction（嵌套观点六元组抽取），首次定义了文档级嵌套观点的结构化抽取范式，突破了现有观点分析研究大多聚焦句子级三元组/四元组抽取、忽略真实文本中普遍存在的嵌套观点现象的局限，专为社交媒体舆情分析、细粒度观点挖掘、公众态度研判等真实业务场景设计。

任务定义
给定一篇由多个句子组成的文档级自然语言文本，要求模型通过理解文档全局长上下文语义、跨句子关系与嵌套层级语义，抽取两类结构化观点信息，同时完成一阶普通观点解析与二阶嵌套观点挖掘两大核心目标：

一阶观点四元组抽取（普通四元组抽取）：识别文本中所有基础的主观观点单元，抽取合法的观点四元组，形式化为 (Holder1, Expression1, Target1, Polarity1)。

二阶嵌套观点六元组抽取：识别文本中所有针对一阶观点的上层主观态度，抽取合法的嵌套观点六元组，形式化为 (Holder2, Expression2, (Holder1, Expression1, Target1, Polarity1))。

数据集描述
嵌套观点六元组抽取赛道采用 NOSE 数据集（中英双语），包含一阶观点四元组、二阶嵌套观点六元组、跨句子实体共指链、情感极性分类等标注信息。中文原始数据采集自微博、X 等主流社交媒体多主题评论文本，英文数据集在中文标注数据集的基础上使用谷歌翻译 API 完成初始翻译，并由具备 NLP 与情感分析背景的专业人员进行人工校对、标注对齐与内容修正，整理为结构化 json 文件。输入输出样例如下：

输入样例

评价指标
本次评测将基于 Opinion Elements F1、Quadruple F1、Sextuple F1、Quadruple & Sextuple F1 四项指标的综合排名进行评定。具体方法如下：首先，分别根据 Opinion Elements F1、Quadruple F1、Sextuple F1 和 Quadruple & Sextuple F1 四项指标对所有参赛队伍进行独立排名；然后，将每支队伍在两项指标中的排名位次相加，总和最小的队伍位列前茅。若两支队伍的排名总和相同，则以 Quadruple & Sextuple F1 的高低作为最终排序依据，得分较高者优先。
